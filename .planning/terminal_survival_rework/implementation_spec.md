# 第四频段整合改动规格

## 1. 实施顺序

改动分成四个可独立验证的版本，不把 UI、主线、校正体和最终战一次性混改。

1. **终端界面版**：四页导航、APP 工具页、文件分栏、任务首页、记录页和弱化调频。
2. **便利工具版**：接入住处、矿物、天气、传送门、特殊信号和要塞方向的真实数据。
3. **生存主线版**：删除计时身体生成的推进作用，改用原版生存节点和成组异象。
4. **要塞终局版**：传送门祭坛、校正体最后阻拦、固定装置和三个结局。

每个版本先完成编译和定向测试，再进入下一版。旧存档迁移在便利工具版开始，在生存主线版完成。

## 2. 终端总体布局

保留 512x256 基准画布、金属外壳和右侧硬件区。主显示区内部改为四页。

### 顶部标签

| 页面 | 建议范围 |
|---|---|
| 主页 | x=42..116, y=45..62 |
| 工具 | x=119..193, y=45..62 |
| 记录 | x=196..270, y=45..62 |
| 文件 | x=273..350, y=45..62 |

客户端新增 `TerminalPage`，页面顺序固定为 HOME、TOOLS、RECORDS、FILES。服务端仍保留现有 SIGNAL/FILES 两种 wire mode：HOME、TOOLS、RECORDS 映射到 SIGNAL，FILES 映射到 FILES。这样不改变终端快照 v5 的 mode 含义。

### 主页

主页只包含三块：

- 当前任务：x=44..348, y=72..119。
- 两个快捷工具：x=44..194 与 x=198..348, y=123..164。
- 最近发生：x=44..348, y=168..196。

当前任务直接复用现有 objective ID、progress、target 的传输形状，但更换任务选择逻辑和玩家文字。首页始终只有一个当前任务。

### 工具 APP 桌面

主显示区采用 3 列 2 行，固定位置不随解锁顺序移动：

| 位置 | APP |
|---|---|
| 第一行左 | 住处 |
| 第一行中 | 矿物 |
| 第一行右 | 传送门 |
| 第二行左 | 天气 |
| 第二行中 | 信号 |
| 第二行右 | 要塞 |

建议单元格：第一列 x=44..142，第二列 x=146..244，第三列 x=248..346；第一行 y=73..128，第二行 y=134..189。

尚未可用的 APP 不画图标、不显示锁。点击 APP 后进入同一页内的详情视图，顶部显示“＜ 工具 / 名称”。从其他标签再次进入工具页时回到 APP 桌面，但正在进行的指路或寻找不会取消。

### 记录页

记录页不再依赖玩家当前调到哪个频率。它按时间显示全部重要记录，并把同一场异象的连续步骤放在一起。第一版先复用现有信号事件，取消按当前 band 过滤；后续生存主线版再增加事件组 ID。

记录可提供“指向发生地点”，但不提供“接受任务”。

### 文件页

分栏发生在主显示区内部，右侧示波器、罗盘和频率屏保持不变。

| 区域 | 建议范围 | 占比 |
|---|---|---:|
| 文件列表 | x=44..133, y=72..196 | 约 30% |
| 分隔线 | x=134..136, y=72..196 | — |
| 正文 | x=137..348, y=72..196 | 约 70% |

规则：

- 每次从其他标签进入 FILES，`selectedFile=-1`。
- 未选择时正文区域只显示“选取文件来查看”。
- 左侧每行一个标题，目录顺序继续使用固定文件目录顺序。
- 左侧列表和右侧正文分别保存 `fileListScroll` 与 `fileContentScroll`。
- 鼠标在左侧时只滚列表，在右侧时只滚正文。
- 切换文件时正文回到第一行，左侧列表不跳动。
- 上下键移动左侧选择；Enter 与单击都直接在右侧显示正文。
- 所有已发现的文件，包括四个碎片文件，都进入左侧列表，不再使用三列卡片和单独碎片目录。
- `encrypted_witness_file` 未解开时，在右侧显示现有线索与缩小后的密码键盘；解开后在同一区域显示完整正文。

文件标题以 2 至 7 个汉字为主，最多 8 个汉字。保留 12 个文件 ID，只改双语标题和内容。`body_mapping_warning` 的 ID 保留，中文含义改为“它在找入口”等新剧情。

## 3. 工具功能与真实数据

### APP 名称

- 住处
- 矿物
- 传送门
- 天气
- 信号
- 要塞

### 住处

- 优先读取玩家床的重生点；没有床时允许点击“把这里记为住处”。
- 显示方向、直线距离和是否在同一世界。
- 点击“开始指路”后占用罗盘；可随时停止或替换。

### 矿物

- 玩家主动选择铁、红石或钻石。
- 复用现有半径 48、分 Tick、有预算的真实扫描。
- 显示方向、大致距离和高度差，不显示方块发光。
- 解除“终端绑定后停止扫描”的旧限制，改为只在功能被校正体暂时关闭时停止。

### 传送门

- 每次真实跨维度后保存到达位置和所在维度。
- 当前世界有记录时显示方向与距离。
- 只使用真实记录，不猜测另一侧传送门的位置。

### 天气

- 第一版只显示当前天气、当前昼夜和距离天黑或天亮的大致时间。
- 在没有可靠预测算法前不显示“即将下雨”。

### 信号

- 汇总现有附近碎片、特殊文件或主线信号。
- 允许从候选列表选择一个目标，复用已有结构碎片选择逻辑。

### 要塞

- 玩家至少投出两次末影之眼后出现。
- 服务端记录样本位置，并用真实要塞位置生成带范围的估算结果。
- 两次样本显示大范围，第三次缩小范围；不直接显示精确坐标。

### 罗盘占用

- 同一时间只有一个工具控制第二根指针。
- 启动新指路时明确提示“已经改为指向……”。
- 停止指路只移除当前方向，不删除住处、传送门或要塞记录。

## 4. 工具数据与网络

保留 `TerminalSnapshotPayload` v5 和 `TerminalNavigationPayload` v4 的字段形状。独立的 `TerminalToolSnapshotPayload` 已由 v1 升到 v2，避免把六个 APP 状态和接收器状态塞进剧情快照。

建议工具快照包含：

- 可用 APP bit mask。
- 当前打开的工具与当前控制罗盘的工具。
- 附近接收器是否可用、当前强度和锁定 Tick。
- 工具是否被暂时关闭以及剩余时间。
- 当前天气与游戏时间。
- 是否记住住处、传送门和要塞。
- 当前末影之眼样本数与要塞距离范围。

动态方向继续由现有导航包每 4 Tick 同步。工具详情由工具快照每 20 Tick 同步。

在现有 `TerminalControlPayload` 中增加动作常量，不改变包结构：

- `SELECT_TOOL`
- `START_GUIDANCE`
- `STOP_GUIDANCE`
- `SELECT_RESOURCE`
- `REQUEST_RESCAN`
- `SET_HOME`
- `MARK_RECORDS_READ`

旧 `SET_AUTO_TUNING = 11` 仅保留编号占位，服务端拒绝执行。

所有 value 必须在服务端按固定枚举范围校验。

## 5. 调频弱化

本节已被 2026-07-21 实机试玩结论取代。最终规格见 `frequency_retirement_spec.md`：

- 删除三个普通频率校准和自动/手动轮播。
- 全局不再使用频段进行内容分类。
- 滑块只在“信号”工具内且附近存在真实异常目标时出现。
- 第四频段改为异常信号层，不是第四个电台。

## 6. 玩家可见任务

沿用单一 objective 结构，建议任务顺序：

1. `set_home`：让终端记住一个经常回来的地方。
2. `bring_iron`：使用终端找到铁并带回来。
3. `prepare_nether`：准备进入下界。
4. `return_from_nether`：带着终端从下界回来。
5. `craft_eye`：制作末影之眼。
6. `record_eye`：投出末影之眼，让终端记住方向。
7. `find_stronghold`：按照方向找到要塞。
8. `find_portal_room`：找到声音最强的房间。
9. `open_altar`：放入剩余末影之眼。
10. `defeat_entity` 或 `protect_anchors`：按真相状态显示最终战任务。
11. `complete`：显示最终结果。

玩家文字禁止出现身体生成、环境连续性、结构修订、防线层数、权限层级等说法。

## 7. 新主线状态与旧存档迁移

持久化 schema 从 v4 升到 v5，新增独立字段，不复用旧字段改变类型：

- `survival_milestone_mask`
- `breach_mask`
- `signature_scene_mask`
- `home_position`、`home_dimension`
- `last_portal_position`、`last_portal_dimension`
- `eye_sample_count`
- `stronghold_position`、`stronghold_dimension`
- `truth_read`
- `altar_started`
- `grounding_anchors_remaining`

旧字段处理：

- `BODY_PROGRESS`、`BODY_STAGE`、`BODY_PROGRESS_ANCHOR_TICK` 保留读取，不再作为新流程计时器。
- 旧存档若已有下界往返或身体进度，则迁移为已经完成住处、矿物和下界相关节点，避免完全退回开局。
- 旧存档即使身体进度 1000，也不再自动在玩家附近生成 Boss；终端改为提示寻找要塞。
- 旧文件 ID、结局字符串和历史日志继续可读。
- `body_mapping_warning` 改写为发现终端正在寻找入口的前任笔记。
- Debug 页面可暂时保留旧字段只读显示，并新增新节点页；完成迁移后再移除“设置身体进度”动作。

## 8. 校正体改造边界

- 第一次标志性事件由生存节点触发，不依赖随机等待。
- 校正体优先处理异常痕迹和终端留下的信号，不主动把玩家当猎物。
- 玩家攻击、阻挡工作或强行恢复被关闭的工具后，校正体才进入敌对状态。
- 工具被关闭时不删除已保存的位置，不夺走物品，不损坏箱子和基地。
- 等待可以自然恢复；主动赶走校正体可以更快恢复。
- 现有造身器官目标改为异常痕迹、信号封壳和最终固定装置。

## 9. 要塞终局

- 最后一个末影之眼触发祭坛流程，不进入末地。
- 未读真相：任务显示“摧毁挡住入口的装置”。
- 已读真相：任务显示“保护固定怪物的装置”。
- 所有路线都需要击败最终实体。
- 三结局只依赖 `truth_read` 与剩余装置数量：
  - 未读真相：未发现真相。
  - 已读真相但装置不足：阻止失败。
  - 已读真相且装置足够：阻止成功。

## 10. 验证要求

### 单元与静态契约

- 四标签和所有互动区域位于主显示区。
- APP 网格为固定 3x2，隐藏 APP 不改变其他图标位置。
- 文件列表约 30%、正文约 70%，左右滚动互不影响。
- 进入 FILES 时无选中项并显示“选取文件来查看”。
- 文件标题中文不超过 8 个汉字。
- 玩家文案不包含已禁止的抽象说法和身体生成文本。
- 非信号工具或附近无真实目标时，服务端必须拒绝调谐。
- 旧 12 个文件 ID、结局 ID 和异象 ID 保持存在。

### GameTest 与客户端场景

- 工具选择由服务端验证，不能选择未开放工具。
- 矿物扫描、住处、传送门和要塞方向均来自真实世界数据。
- 替换罗盘目标不会删除已保存地点。
- 文件页空白初态、选择、左右滚动、密码和完整正文均有截图与断言。
- 普通路线和速通路线都能触发规定的核心场景。
- 旧存档迁移后不会自动生成最终 Boss，也不会丢失已发现文件。
