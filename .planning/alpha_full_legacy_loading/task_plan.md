## 已做决策
| 决策 | 理由 |
|------|------|
| 同时修复启动遮罩与进界加载页 | 用户要求 Alpha 启用后“全面”采用旧版加载界面，两条路径都存在首帧时序窗口 |
| 复用内置 Golden Days Base 的 `mojangstudios.png` 并通过固定类路径纹理加载 | 保持现有后半段旧版视觉一致，且避免资源管理器在约 50% 应用阶段再次替换纹理 |
| 首次崩坏期间不启用持久旧版启动策略 | 保留既有“原版前奏 → 崩坏 → 旧版恢复”的一次性叙事 |
| 不修改 `options.txt` 或 Alpha 状态 schema | 该问题是表现时序，不应扩大持久化边界 |

## 遇到的错误
| 错误 | 尝试次数 | 解决方案 |
|------|---------:|---------|
| 首轮宽泛搜索包含大型资源包并被输出截断 | 1 | 后续限定 Java、测试和具体加载类 |
| `javap` 不在 PATH | 1 | 使用已安装 JDK 21 的绝对路径读取 Minecraft 字节码 |
| 定向 Gradle 测试在已成功编译后被无关终端重构的并发中间态阻断 | 1 | 不修改用户的终端工作；改用首次成功编译产物直接运行 JUnit Console 定向方法 |
| 独立运行 `alpha-relaunch` 时测试运行目录被清理，持久 Alpha/告示状态未保留，套件在等待标题页时超时 | 1 | 先核对状态键和套件入口，再以明确的二次启动夹具状态运行；不把该夹具问题误判为加载实现失败 |
| 文件规划技能的 `check-complete.ps1` 因脚本自身乱码产生 PowerShell 解析错误 | 1 | 不修改技能安装目录；改用只读搜索确认本作用域计划无未完成复选框且三个阶段均为 complete |
