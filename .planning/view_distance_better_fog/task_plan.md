# 任务计划：锁定 3 区块视距并整合 Better Fog

## 关键问题
1. Forge/NeoForge Better Fog 的注入点能否在本项目 Fabric 1.21.x 渲染 API 上等价实现？
2. “视距固定 3”应同时约束客户端区块缓存、雾起止距离和设置持久化的哪些入口？
3. 外部源码是否有允许复制、修改和再分发的明确许可证？

## 已做决策
| 决策 | 理由 |
|------|------|
| 使用独立 `.planning/view_distance_better_fog/` 作用域 | 不覆盖仍在进行的终端生存主线历史计划 |
| 先审计许可证和版本，再复制任何外部代码 | “技术可移植”与“可合法打包”必须同时成立 |
| 不复制 Better Fog 源码；按公开可观察的功能目标独立重写轻量环境雾 | 源项目为 All Rights Reserved，且 Forge 1.20.1 实现与当前 Fabric 1.21.11 不兼容 |
| 视距选项固定保存为 3、有效值最多为 3、视频控件禁用并明确标记 | 同时覆盖配置文件、运行时与设置界面，服务器若低于 3 则尊重其更低上限 |
| 雾增强仅作用于大气路径，水/熔岩/细雪和状态效果继续由原版环境雾优先 | 不削弱危险提示和药水效果，也不会扩大可见距离 |
| 现有红雾在新环境雾之后叠加，继续使用更近的 6/28 格边界 | 保留既有异象强度与测试契约 |

## 遇到的错误
| 错误 | 尝试次数 | 解决方案 |
|------|---------:|---------|
| 文件规划恢复脚本命中 Windows Store 的 Python 占位符 | 1 | 不重复调用系统 `python`；改用工作区依赖运行时或人工核对计划与 Git 状态 |
| 当前 PATH 只有 Oracle Java 跳转器，且首次选到未映射 Minecraft JAR | 2 | 通过 Gradle `javaToolchains` 定位 JDK 21，并改用 Loom named clientonly JAR；字节码审计已成功 |
| PowerShell 中一次翻译键 `rg` 表达式因引号转义失效 | 1 | 改为直接读取两份语言 JSON 的 UTF-8 尾部后精确打补丁 |
| 首轮客户端编译发现新增注入方法漏导入 `CallbackInfo` | 1 | 已补入正确 Mixin callback 类型，未改实现方案 |
| 一次只读客户端测试盘点因末尾 `rg` 无匹配返回 1 | 1 | 输出已包含所需入口；后续用精确方法名和补丁上下文继续 |
| 首轮 mainline 在视距初始化断言处失败 | 1 | 已确认保存文件为 3 且 Mixin 加载成功；增加 stored/effective 诊断值以定位后置限制 |
| 诊断命令尝试读取不存在的可选 GameTest properties 文件 | 1 | 不重复读取；使用现有 options、日志和字节码继续 |
| mainline 后置流程将内存视距从 3 写为 5，底层有效值仍为 3 | 2 | 在 `OptionInstance.set` 唯一写入口识别视距实例并把任何请求值改写为 3 |
