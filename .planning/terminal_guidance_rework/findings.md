# 发现与决策

## 用户需求
- 导航列表和其他工具内部不能一次显示全部可用/未来内容；应按主线进度先给关键项，并在服务端判断玩家卡住时逐步追加提示。
- 新增终局要求：要塞末地传送门改为祭坛入口，进入末地后用自定义 BOSS 替换末影龙。
- 塔顶装置保留时会缓慢治疗 BOSS、但同时让 BOSS 更慢更弱；装置越少 BOSS 越强，全部摧毁时应接近无法战胜。
- BOSS 血量按实际参与玩家数量动态调节。
- BOSS 战使用伪倒计时：倒计时归零且 BOSS 存活就锁定“阻止失败”，但战斗仍继续；归零前击杀则为“阻止成功”。
- 档案需要提前警告玩家不要破坏塔顶装置；玩家可见称呼统一改为“稳定锚”，不出现“末影水晶”。
- 保留调频滑块；前期拖动只有音效，不改变任何玩法或终端状态。
- 第一个任务改为采集 7 个原木；后续节点选择大多数生存玩家为了击败末影龙必须经历的行为。
- 任务文案有“采集数据”的感觉，但要直白易懂。
- 任务进度条平滑更新。
- 首页按玩家当前任务与状态智能推荐工具；选择后以简约实时详情替换中间两张卡片；返回时回首页。
- 修正“文件”命名；将“信号”改成“导航”。
- 导航前期指向最近村庄等结构，中期加入支线结构，并以淡红渐变闪烁显示“[不稳定信号源]”。
- 六个工具图标使用原版像素风与终端配色。
- 工具不应全部开局即用；导航、钻石寻找等应按合理阶段解锁。
- 固定 3 区块视距使终端导航成为关键可达性系统。

## 已知项目背景
- 当前客户端有效视距已固定为 3 区块，视频设置控件被禁用。
- 现有终端为首页、工具、记录、文件四页，并有六个工具、独立工具快照和单一罗盘导航占用。
- 旧固定频段已退场，只保留近场异常接收器；本轮不能重新引入可跳过的三频校准门槛。
- 当前工作区没有 Git 提交且项目文件整体未跟踪；本轮不清理、不回退、不提交。

## 待审计
- 当前任务链的真实节点与进度计数来源。
- 首页两张卡片与工具详情返回状态的具体实现。
- “文件”命名错误的实际翻译键与页面语义。
- 六个工具现有图标是 PNG 资源、物品贴图复用还是代码绘制。
- 原版结构定位 API、扫描预算与已经保存的结构坐标。

## 初步代码定位
- 主线任务由 `SurvivalMilestone` 与 `StoryProgressService` 选择；终端客户端通过 `TerminalSnapshot` 接收 objective ID、当前值和目标值。
- 六工具由 `TerminalTool`、`TerminalToolService` 与工具快照协议驱动；页面渲染和全部点击状态集中在约 1555 行的 `TerminalScreen`。
- 终端图标没有独立命名资源出现在首轮文件列表中，需继续确认是代码绘制还是复用原版物品渲染。
- 相关视觉入口集中在 `M0ClientGameTest`；生存主线服务端入口集中在 `SurvivalReworkGameTests`。

## 主线与工具现状
- 当前任务链是住处→铁→下界准备→下界往返→末影之眼→两次投眼→要塞→传送门房→自定义祭坛/最终实体，并没有末影龙节点。
- `EndPortalBlockMixin` 会无条件取消末地传送门的实体进入逻辑；若终点改成击败末影龙，必须同步解除该阻断并迁移原自定义祭坛入口，否则新任务不可完成。
- 生存进度目前每秒扫描背包与已保存位置；没有“挖掘原木计数”。可使用服务端玩家 `BLOCK_MINED` 统计对原木标签内方块求和，使拾取/赠送原木不能伪造“挖掘 7 个”。
- 当前六工具中住处、矿物、天气、信号开局全部可用；传送门和要塞按是否已有坐标出现。矿物资源选择没有逐资源解锁，玩家开局即可选择钻石扫描。
- 当前“信号”只导航四组剧情碎片并承载近场接收器；其结构导航状态复用 `NavigationState`，可以在保留异常近场调谐的同时扩展为普通结构目标。
- 工具快照协议 v2 只有 `availableToolsMask`，不足以同时表达“可见但未解锁”和原因；本轮需要增加 unlock/usable 状态或等价字段，不能仅用客户端灰显猜测。
- 终端现有自定义祭坛路线与用户指定的末影龙终点直接冲突，这是本轮最关键的玩法迁移边界。

## UI 与文案现状
- 首页任务下方固定写死为“住处”和“矿物”两张快捷卡，不读取 objective 或玩家状态；点击后把页面切到工具详情。
- 工具详情的返回按钮总是回工具网格，因此从首页进入也无法回首页。可让首页保持 `HOME` 页面，用一张跨两栏的实时详情卡替换两张推荐卡；工具页仍保留完整详情和网格返回。
- 任务进度条直接使用最新快照比例绘制，数值变化会瞬间跳变；需要增加客户端显示值与目标值之间的平滑插值，并在 objective ID 变化时从 0 或合理起点重新过渡。
- 六个工具目前共用同一个代码绘制的十字占位符，没有独立 PNG；适合改为六套代码原生 9×9/像素矩阵图标，缩放时保持整数像素和终端配色。
- 中文 `terminal.thefourthfrequency.tab.files` 实际值为 `[文件]`，英文为 `[FILES]`。内部页面承载发现/解锁的叙事资料，玩家可见名称改为“档案 / ARCHIVE”更符合语义；wire ID 和文件 ID 保持不变。
- 当前调频滑块只有在“工具 > 信号”且附近真实异常目标存在时才绘制/可交互。用户要求“保留滑块且前期拖动只有音效”，因此需要把“显示/机械交互”和“服务端生效”拆开：前期允许本地拖动与声音，但不发送 TUNE、不中断/启动任何导航或剧情状态。

## 冻结设计
- 所有工具采用渐进披露：默认仅首要目标；当前任务在线无进展约 2 分钟开放一个备选，约 5 分钟开放支线；异常入口还要求玩家至少进入过下界；开发加速模式为 10/30 秒。
- 停滞只累计玩家在线 Tick；任务 ID 或数值进度变化立即清零。已选中的结构/异常目标在后续收束时继续显示。
- 矿物页不再渲染未来资源的锁定按钮：原木后只有铁；取得铁且停滞一层后补充钻石，二层再补充红石；返回下界后钻石稳定开放。
- 首页默认只给首要推荐，第二卡显示等待更多数据；停滞一层或夜间无住处等紧急状态才给第二建议。
- 终局修订优先于先前“击败末影龙”字面实现：完整生存路线仍通往末地，但末地目标在玩家进入后更新为自定义 BOSS。
- 要塞传送门承担祭坛入口，不再生成主世界新祭坛；`EndPortalBlockMixin` 继续退出 Mixin 列表，确保末地可达。
- 使用既有 `MisreadBodyEntity` 作为最终 BOSS，并将旧主世界祭坛逻辑保留为旧存档恢复/调试路径。
- “稳定锚”按存活数量应用单调难度档位；治疗量必须较低，减速、攻击降低和承伤提高的收益必须足以让保留更多锚始终更容易。
- 零锚档采用极高减伤、极快移动和高攻击，但不设置绝对无敌，防止世界永久软锁。
- 倒计时只在末地有存活参战玩家时推进并持久化剩余时间；掉线、重启或离开末地不能重置计时。
- 失败一旦在零秒锁定，之后击杀 BOSS 只完成战斗，不改写失败结局。
- 新任务链：挖掘原木 7 → 获得铁 → 进入下界 → 收集烈焰棒 6 → 返回主世界 → 制作末影之眼 → 真实投眼 2 次 → 接近要塞 → 启动祭坛入口 → 进入末地 → 击败异常主体。
- 原木进度从 `Stats.BLOCK_MINED` 对 `BlockTags.LOGS` 求和；烈焰棒进度用拾取统计、当前背包和已制作烈焰粉/末影之眼的等价量取可靠上界，并写入持久记录，避免快速合成后回退。
- 末地进入通过真实维度切换记录；末影龙完成通过 Fabric 服务端生物死亡事件记录，并在玩家进入已经屠龙的末地时兼容补记。
- 旧自定义祭坛不再注册新门框交互，`EndPortalBlockMixin` 从 Mixin 列表移除；旧服务类继续处理已经存在的祭坛状态和调试夹具。
- 首页推荐由服务端根据 objective、活动导航、住处、时间和工具状态给出两个槽位；客户端不自行推测权威解锁。
- 导航结构扫描只在玩家明确选择目标时执行并缓存到现有 `NavigationState`，不做每 Tick 全图定位。
- 结构标签使用 1.21.11 已确认的 `StructureTags.VILLAGE/MINESHAFT/RUINED_PORTAL`；试炼密室、下界要塞和堡垒遗迹使用本 MOD 自有结构标签引用原版结构 ID。
- 本轮保持项目既有约束：只运行编译/打包验证，不主动执行 JUnit 或 GameTest；测试源码会同步更新并保证可编译。
