# 当前末地祭坛与终端引导规则

本文覆盖旧 Alpha 报告中的终局与终端工具描述；旧字段、旧祭坛代码和第三结局枚举只为存档恢复与调试保留。

## 生存任务链

终端只承认真实服务端行为：挖掘 7 个原木 → 取得铁 → 进入下界 → 收集 6 根烈焰棒 → 返回主世界 → 制作末影之眼 → 真实投掷两次 → 接近要塞 → 启动要塞祭坛入口 → 进入末地 → 击败异常主体。文案使用“样本、记录、方向验证”等直白数据采集语气，不要求玩家理解隐藏系统名。

## 稳定锚

玩家可见文本不得把塔顶装置称为末影水晶，统一称为“稳定锚”。档案《进入祭坛前》会提前给出三条事实：

- 稳定锚会缓慢修复目标；
- 它们对目标速度、攻击性、适应频率和抗性的抑制远强于这点治疗；
- 每损失一个难度档，目标都必须更难对付；全部清空后，现有武器几乎无效。

稳定锚档位为 8+、5–7、2–4、1、0。归零档不设绝对无敌，以免存档永久软锁，但只承受 3.5% 的正常伤害，并获得最高移动、攻击和适应频率。

## 多人与结局

BOSS 基础最大生命为 360；本轮战斗见过的最高同时参战人数每增加一人，最大生命增加 72%。人数上升时保持当前生命百分比；人数减少不会降低最大生命，避免通过队友退出压低血量。

正常干预窗口为七分钟；开发加速模式为三十秒。计时只在末地中央区域内存在存活参战玩家时推进，并持久化剩余时间，离开、掉线和重启都不能重置。

- 窗口归零前击杀：`prevention_succeeded`。
- 窗口归零时 BOSS 存活：立即锁定 `prevention_failed`，但不移除 BOSS、不传送玩家，也不终止战斗。
- 失败锁定后再击杀：战斗完成，失败结局不被改写。

## 终端交互

- 首页首要推荐由服务端根据任务、里程碑、时间、住处与当前导航计算；第二项默认留作“等待追加建议”，只在夜间缺少住处或目标停滞后出现。
- 首页点开的工具在原位替换两张推荐卡，并以实时快照更新；返回只关闭该卡，仍留在首页。
- 六个工具始终可见；未解锁工具可查看条件，但所有服务端动作继续由权限掩码拒绝。
- 原木任务完成后开放矿物与导航，但每页默认只披露当前最相关的一个目标。取得铁后若推进停滞才加入钻石分析，长时间停滞再加入红石支线；返回下界后钻石分析成为稳定可用项。真实下界穿越后开放传送门；制作末影之眼后开放要塞估算。
- 调频滑块始终可拖动。无有效近场异常时只播放声音和本地视觉反馈，不发送 `TUNE`。
- 导航默认只提供当前主线结构；在线且目标无进展约 2 分钟后追加备选，约 5 分钟后才追加支线。缓慢淡红呼吸的 `[不稳定信号源]` 同时要求玩家至少进入过下界。任何进度变化都会重置停滞计时，离线不累计，已选目标继续保留。
