# 《第四频段》世界观圣经

本文描述 `0.2.1-beta` 的现行叙事边界。数值与测试以 [世界接口祭坛与终端指南](end-altar-and-terminal-guidance.md) 和 [测试与验收](testing.md) 为准。

## 核心命题

世界并非突然被外来怪物入侵。它一直在用玩家熟悉的方块、菜单、声音和规则维持一层可理解的外观，而“第四频段”是这层外观无法完全压住的通信噪声。

终端不是任务列表的外壳，而是世界用有限带宽解释自身的接口。它先学会记录玩家，再学会回答；当回答越来越准确，世界也越来越难继续假装一切正常。

## 不可动摇的事实

- 零号站与终端是故事入口，终端是玩家唯一稳定的解释工具。
- 生存主线仍建立在原版行动上：木材、铁、下界、烈焰棒、末影之眼、要塞和末地。
- 四份破损文件来自对原版世界遗迹的调查，不依赖额外生成的“四处设施”。
- 异象不是随机彩蛋合集；它们按 1—5 层逐渐从环境误差走向界面和规则层。
- 校正者以五种个人形态学习玩家的声音、路线、预测、空间和界面；它是世界接口学习“如何使用身体”的前置证据，不是最终敌人的低级血条版本。
- 正式追逐发生在与其他玩家隔离的私人校正层。孤独感来自观察者被暂时从共享现实中摘除，而不是让其他玩家假装看不见。
- 最终战是多人共同提交解释权的仪式：每个参战者都必须交出自己的绑定终端。

## 体验曲线

### 1. 可信的旧版本

第一次进入世界时，客户端完成一次 Alpha 风格降级：资源包按 Programmer Art → Golden Days Base → Golden Days Alpha 管理，加载界面与版本文案把熟悉的游戏拉回一个不完全真实的 “Minecraft Alpha 1.0.0”。

它应首先让人怀疑版本和记忆，而不是立刻宣告恐怖主题。

### 2. 可重复的误差

异象从声音、光线、方块和记录上的小偏差开始。服务器只决定“发生了什么”，客户端决定“目标玩家如何感知”；多人可以共享发现，却不必共享同一幻觉。

终端的语言应保持短、冷静、像一台努力压低噪声的仪器。它可以不完整，但不能用全知旁白替玩家下结论。

### 3. 世界开始记录玩家

住所、矿物、传送门、天气、导航和要塞工具逐渐从扫描器变成证词。三次真实末影之眼投掷尤其重要：世界接口不是凭空给坐标，而是在承认玩家已经证明过方向。

FILES 中的隐藏文件把零散现象连接成同一套身体/接口映射，但保留解释空间。玩家应逐渐意识到：所谓“异常”也许只是世界无法继续用原版词汇描述自己的时刻。

### 4. 私人校正层

普通异象先以安全方式演示校正者下一形态的规则；终端随后把观察转译成可执行的短警告。正式追逐才把玩家送入与当前维度对应的私人镜像。

镜像不是另一个可以开采的世界。它复制玩家附近的现实并沿逃跑路线继续展开，但容器、战利品、其他玩家和真实进度都被剥离。玩家可以开路和临时放置，却不能从虚假地形中带回资源。这里的叙事边界是：

> 世界不是真的，但受到的伤害是真的。

校正者五形态不是单纯加速：

- 寻声者把重复行动解释成位置；
- 定向者把常走路线解释成意图；
- 截断者把移动趋势解释成未来出口；
- 越界者不再接受墙壁和天花板作为边界；
- 接口校正者开始伪造坐标、文字和方向，只有连续波形仍可信。

玩家提前推进主线不会被连续追逐惩罚。世界只保留下一次校正机会；每个形态都必须先被看见、被终端解释，再进入正式对抗。

### 5. 集体交出终端

末地祭坛贴合原生主岛，因为终局必须发生在玩家认识的世界里。20 座惰性门像未启用的出口，10 个稳定锚像世界仍在拒绝崩溃的局部承诺。

冻结参战名单和逐人提交终端具有叙事含义：没有任何一个客户端能单方面代表所有观察者。提交前可以撤回，提交后则进入共同承担的事务。

## “第四频段·世界接口”

世界接口不是传统意义上的肉身。三种形态是同一接口在受压后逐步放弃可读性的表现，形态只能向前，不能因治疗退回。

稳定锚也不是简单的“先拆塔”。它们会治疗目标，却同时让它更慢、更容易受伤、出招更迟。拆除会换来时间上的立即损失和更危险的接口；这种选择应让玩家争论，而不是照着唯一攻略执行。

九类行动围绕“世界怎样解释玩家”展开：

- 激光、能量球与箭矢反射把空间和弹道变成协议回声。
- 抓取砸落与抓取投掷暂时夺走位置解释权，但必须给出预警和安全落点。
- 精神侵袭攻击感知，而非伪装成不可追踪的随机伤害。
- 武器托管和凝视清栏暂时接管物品叙事；所有物品都进入恢复账本，恐惧来自失去控制，不来自暗中吞物。
- 强制驱逐把连接本身当作攻击，但不会选取集成服务器房主，也不能让 LAN 世界被一个客户端演出终止。

世界接口能给末地留下永久伤痕，但伤痕必须有限、可解释并避开原版关键结构。它不能摧毁稳定锚，也不能借“Meta”之名无上限地修改玩家世界。

## 两种结局

### 成功：在解释坍塌前切断接口

玩家在倒计时结束前击败世界接口。祭坛开放返回出口，原版终末之诗的通道被用于呈现本次成功诗篇。只有诗篇完成且玩家真实返回主世界后，长期受限的视距才从 3/6/12 解锁到 16。

这不是回到一切发生前：战场伤痕仍在，标题页结局锁也承认这段世界已结束。

### 失败：接口跨过客户端边界

崩塌先完成时，世界接口进入逃逸解析。普通客户端完成固定演出后保存、断开并正常关闭。若失败者是已发布 LAN 的房主，服务器必须继续存在；只有房主本地看到整个世界变成缺失材质，客人看到的方块和规则不变。

失败不是破坏操作。恐怖效果属于客户端演出和无损标记，不允许改写 `level.dat`、区块或玩家数据。

## 结局锁、F8 与重玩

两种结局都使标题页只禁用三个进入世界的入口：单人游戏、多人游戏和 Realms。设置、辅助功能和退出仍然存在，这是玩家安全退出和恢复的边界。

F8 只有在持久结局锁存在时才打开确认。确认恢复 MOD 持有的桌面、记事本、窗口和资源包状态，然后正常关闭。再次启动时，准确对应的本地结局存档通过 `.thefourthfrequency-corrupted` 标记显示为“存档已损坏”，新一次故事必须从新世界开始。

`-Dthefourthfrequency.safeMode=true` 是演出中断后的恢复后门，不是叙事捷径。

## 写作与视听边界

- 终端文案短句优先，先给可操作信息，再给不完整解释。
- 不用虚假的系统错误替代规则提示；重要控制必须有清晰预警。
- 恐怖来自熟悉系统被重新解释，不来自无提示删档、吞物或无限制改桌面。
- 高对比、闪烁和峰值音量受首次安全告知与配置边界约束。
- Meta 效果只能改变 MOD 明确持有或可恢复的本地状态；多人服务端权威数据不受客户端幻觉污染。
