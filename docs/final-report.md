## 已实现范围

### 终端与生存主线

- 世界/终端 schema v7，主快照 v6，工具 v3，导航 v5。
- HOME / TOOLS / RECORDS / FILES 四页界面与六个服务端权威工具。
- 从木材、下界、烈焰棒、两次真实末影之眼投掷到要塞和世界接口的完整目标链。
- 完整日记常驻 FILES；四份破损文件共享发现、个人独立阅读，第一位 4/4 阅读者只创建一次共享裂缝。

### 异象与连续性

- 16 种可触发异象，覆盖 1—5 层；旧实验 ID 只保留为兼容名称映射。
- 服务器权威触发/进度与客户端私有呈现分离。
- 旧 `MisreadBodyEntity` 仅保留在前置叙事，不再承担最终 BOSS。

### 世界接口终局

- 原生末地主岛、11×11 贴地祭坛、20 座惰性门和 10 个稳定锚。
- 1—8 人冻结名单、逐人终端托管、撤回、回滚和原子提交。
- `600 × 人数` 生命、三种单向战斗形态、10 分钟崩塌、九类行动和物品恢复账本。
- 2048 格永久伤痕总预算、每 Tick 8 格，以及原版关键结构/方块实体/免疫标签保护。
- 同 Tick 超时优先；成功与失败都有确定的解析、出口和完成状态。

### 结局与客户端状态

- 3×3 返程出口接入原版 `showEndCredits → WinScreen → PERFORM_RESPAWN`，按结果和语言替换本次诗篇。
- 成功返程后把维度视距锁 3/6/12 永久解锁为 16；失败不解锁。
- 普通失败保存、断开并正常关闭；已发布 LAN 房主保持服务器开放，缺失材质只影响房主客户端。
- 两种结局都建立本地锁；F8 提供带确认的完整恢复，而不是 Meta 开关。
- 结局存档仅通过 `.thefourthfrequency-corrupted` 标记隔离，不破坏原版世界数据。
