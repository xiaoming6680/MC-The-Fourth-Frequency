# 《第四频段》Alpha 0.1 最终报告

## 交付结论

Alpha `0.1.0-alpha.2` 已完成 R42–R49 异象系统重构、R55–R60 客户端异象自动化、R74–R99 体验/终端修订、R100–R105 原版结构碎片主线与 R111–R115 弱引导流程收口：目录固定为 16 个可触发单异象，新世界不再生成五座前人设施。

当前最新界面使用协议 v5 的 HOME / TOOLS / RECORDS / FILES 四页终端与工具快照 v2；异象使用 v3 的启动/阶段/完成载荷，Debug 状态仍为协议 v2。持久化 schema 为 v5，活动异象不写入存档，旧频段、校准、自动调频和复合字段仅作兼容读取。

## 已实现内容

- 唯一零号站、每玩家唯一终端、世界账本权威与安全恢复。
- HOME / TOOLS / RECORDS / FILES 四页终端、六项真实生存工具、聚合记录与同维度导航；不存在玩家可见固定电台、三频校准、自动轮播或全局调频。
- 四个隐藏类别的碎片调查：每类从多个原版结构中保存 2–3 个不重复坐标；只有玩家身处真实候选范围且打开“工具 → 信号”时，近场接收器才开放，锁定后触发私人短异常，文件按世界共享；不依赖箱子、战利品或特定方块。
- 四组候选统一显示在信号工具中，不按历史频段分散；记录页聚合全部事件并在进入时统一标为已读，文件页才显示“前人留下的文件 / 碎片1–4”，集齐后读取完整文件。关键记录保留标题右侧 `＋/－` 折叠、12 像素行距、缩进圆点和精简坐标句式。
- 趋向群、返工体和两种稀有可恢复空段。
- 16 种按五级分布解锁的单异象、每玩家单活动实例、服务端权威现实变化、目标客户端私有幻觉与统一租约恢复。
- 0–5 世界崩坏；标题、图标和选择性纹理从 1 级开始，3 级起可退出全屏，断线/归零/成功结局恢复窗口快照。
- 主世界/下界规则裂缝、换维度后继续记录同一玩家、0–1000 在线身体成形进度与多人私人异象。
- 利用玩家长期习惯改变形态的最终肉身、Boss Bar、环境吸收成长、正常伤害击杀、终止钉可选削弱与三个互斥永久结局。
- 固定事件驱动、可随时关闭、失败可降级的 Windows 可信 Meta 演出。
- 第一次进入单人或多人世界后的本地安全告示、不可绕过确认与跨世界一次性持久化。
- 有峰值限制与字幕事件的声音层、六态终端图标、双语界面和小窗口交互路径。

## 数据与兼容性

- 世界与终端 schema：v5，生产迁移链 0→1→2→3→4→5。
- 终端主快照：协议 v5。
- 终端工具快照：协议 v2。
- 终端导航：独立协议 v4，打开立即发送并每 4 Tick 刷新。
- 异象生命周期：协议 v3；Debug 状态：协议 v2。
- 永久结局：版本 v1，单向不可覆写。
- 旧设施、密码、裂缝、校准位图、自动调频和日志频段字段支持兼容读取；已有旧设施的存档仍可读取，新世界不再分配设施。未来 schema 明确拒绝，避免旧版本损坏新存档。

## 性能与安全

零号站、原版结构候选定位、旧存档设施、下界裂缝、校正系统、趋势采样和资源扫描均有固定每 Tick 上限，并只处理已加载区块、节流定位或空间索引。未发现无限生成、每 Tick 全世界扫描、物品吞失、存档损坏或持续服务端卡顿。

服务端不接受客户端任意文本、路径、命令或世界操作。Meta 协议只传固定数字事件；Windows 实现只操作自有窗口状态、当次新建目录、自有文件与持有句柄，异象时降级为游戏内提示。

## 已知非阻断限制

- 声音混音舒适度、字幕阅读速度、通知外观和多屏/DPI 窗口体验仍需在目标玩家硬件上人工复核。
- 本次宿主为 Windows；Linux/macOS 的降级逻辑已有单测与代码边界，但尚未做对应实机客户端体验验收。
- 多人规则由多个服务端测试玩家覆盖；正式发布前仍建议用两台真实客户端检查网络延迟下的体验。
- 当前声音和部分方块/物品视觉可合法运行并通过资源契约，但仍有独立录音与高保真美术的润色空间。
- 工作区当前 `main` 尚无提交；所有工程文件均为未跟踪内容，本轮未提交、推送或创建 PR。
