# Beta 0.2.1 人工验收与游玩说明

本清单面向 `0.2.1-beta` 候选包。自动化结果与命令见 [测试与验收](testing.md)，异象/追逐规则见 [异象、终端形态与个人追逐](anomalies-and-pursuits.md)，世界接口精确数值见 [末地祭坛与终端指南](end-altar-and-terminal-guidance.md)。

## 验收前准备

- 使用 Minecraft 1.21.11、Fabric Loader 0.19.3、Fabric API 0.141.4+1.21.11 与 Java 21。
- 安装 `thefourthfrequency-0.2.1-beta.jar`，首次验收优先使用新存档。
- 末地结局会给当前存档建立本地隔离标记；正式跑结局前先备份存档。
- 默认关闭开发加速：`pacing.developerAcceleration=false`。
- 需要从中断的结局事务恢复时，以 `-Dthefourthfrequency.safeMode=true` 启动一次。

## 首次安全告知

1. 清除本地安全告知版本文件后进入标题页。
2. 确认 v3 告知在第一次 `TitleScreen` 出现，包含强闪烁/高对比、音量与不可逆存档演出的说明。
3. 确认同一告知版本只显示一次；确认状态写入 `config/thefourthfrequency-safety-notice.version`。
4. 在尚未完成结局时按 F8，确认不会切换 Meta，也不会弹出结局重置框。

## 新世界与终端

1. 新建生存世界，确认零号站与唯一终端流程可达。
2. 终端应只有 `HOME / TOOLS / RECORDS / FILES` 四个玩家页面；网络线协议仍只使用 `SIGNAL / FILES`。
3. 首项任务应为“认识终端”；只有依次实际选择过四个顶部标签后才完成。任务完成后手持终端切到未读式外观，点击原尺寸任务卡领取右上角显示的奖励后恢复。
4. 六个工具应依次可用：住所、矿物、传送门、天气、导航、要塞。
5. 核对主线目标顺序：

   `12 份任意原木/木板 → 4 份铁样本 → 下界 → 8 根烈焰棒 → 返回主世界 → 合成 4 只末影之眼 → 记录 3 次真实投掷 → 接近要塞 → 进入末地 → 击败世界接口`

6. 真实末影之眼投掷不足三次时，要塞工具不能提前给出完整定位结论。
7. 完整日记应从一开始常驻 FILES，初始标题全部为动态乱码；每发现一份破损文件，标题恢复约 25%。
8. 四份破损文件只在发现后出现，名称始终为暗淡斜体“破损的文件”，正文约三成可读；首次打开分别推进 25%，全部读完才解锁完整日记。
9. 多人中发现状态共享、阅读状态独立；第一位 4/4 阅读者创建裂缝，后来玩家读完时不得重复生成或移动裂缝。

## 异象与记录

1. 检查目录为 16 种可触发异象、阶段为 1—5；`disconnected_base`、`watcher_orbit`、`rework_probe`、`hostile_echo` 只能作为历史/调试名称兼容。
2. 新终端第一次异象应安排在 4–7 分钟；普通间隔按阶段落在 8–14、8–13、7–12、6–10、5–9 分钟。
3. 即使提前获得铁，实际阶段也不得跳过阶段 2；每次升级要求当前阶段至少 20 分钟合格在线时间和 2 次成功异象。
4. 最近三次成功异象不得再次入选；升层后优先观察到新阶段内容。
5. 窗口脉冲、频道接管、桌面在场触发后，20–30 分钟内不得再次出现强接口异象。
6. 多人中个人视觉、声音和输入效果只作用于目标客户端；观察者定向、门扉级联、经历缺口等共享型服务端效果允许附近玩家看见结果。
7. 默认按 M 打开 Debug 工作台；它应只显示总览、主线、异象、终局四个模块。服务端仍需对调试动作执行权限和上下文校验。

## 个人校正者追逐

1. 分别用挖矿、累计探索 128 格、方块实体/战利品交互、建造和交易建立第一形态活动证明；五条路线均不得要求固定住宅。
2. 第一场前必须先成功完成至少一次异象；随后终端给出寻声者应对警告，并至少留出 90 秒准备时间。
3. 核对五个许可门槛：早期活动、进入下界、烈焰棒并返回、3 次真实投眼、找到要塞。
4. 玩家提前跨过多个门槛时，始终只面对下一实际形态；不得跳形态或连续补发多场追逐。
5. 核对时长与应对：寻声者 60 秒、定向者 75 秒、截断者 85 秒、越界者 95 秒、接口校正者 110 秒。
6. 成功后下一场正式追逐需等待 20–30 分钟；捕获不得死亡或掉落，保持待追逐并在 5 分钟后重试。
7. 捕获、追逐成功和主动断线都必须返回原维度安全位置；来源位置被占用时不得覆盖真实方块。
8. 追逐期间普通异象、主线采样、导航、世界衰败和世界接口服务不得从镜像维度写入真实进度。

## 私人镜像与多人隔离

1. 两名玩家同时进入不同形态追逐，确认分别占用不同镜像槽位，互相看不见玩家、校正者、方块修改和掉落物。
2. 第三名玩家在两槽已满时只保留待追逐，不得抢占或传送失败。
3. 入场初始地形应匹配来源位置的 5×5 区块、垂直 ±48 格；水平跑出初始范围后应持续加载新复制区块，不得在 30 格处折返。
4. 极端传送到未复制区块时，只允许短暂停在最近安全位置并暂停追逐倒计时；区块就绪后继续。
5. 镜像挖矿不得获得方块、经验或环境开路耐久损耗；红石、传送门、容器、TNT、床和流体交互应被拒绝或净化。
6. 放置普通建筑方块后结束追逐，确认物品只退款一次；背包已满、断线和重启后仍能由恢复账本交付。
7. 当前版本只允许从主世界和下界开始追逐；末地与模组维度不得正式触发。

## 祭坛与集体仪式

1. 进入末地后，在原生主岛核对贴地 11×11 祭坛、20 座惰性折跃门与 10 个稳定锚；不应出现大型人工战斗平台。
2. 仪式开始时冻结当前在线的非旁观者名单，支持 1—8 人；名单中每人必须提交自己的有效绑定终端。
3. 全员提交前应能撤回或取消；名单变化、重载不确定或事务失败时必须返还终端。
4. 全员提交后应原子提交托管，进入召唤与战斗流程。

## 世界接口战斗

- 最大生命值为 `600 × 冻结参战人数`。
- 战斗形态按生命值单向推进：第一形态 `>70%`，第二形态 `>35%`，第三形态 `≤35%`。
- 崩塌计时为 12000 Tick（10 分钟）。只要任一冻结成员在线，计时就继续；全员离线时暂停。
- 存活稳定锚每座每秒回复目标最大生命的 0.01%。拆锚会让目标更快、更频繁且更难受伤，同时每座把崩塌进度推进 600 Tick（30 秒）；目标自身不能拆锚。
- 九类行动都应能在定向验收中出现：激光扫射、能量球、抓取砸落、精神侵袭、武器托管、抓取投掷、凝视清栏、箭矢反射、强制驱逐。
- 武器托管和清栏掉落必须进入恢复账本；强控制不能并发套在同一目标上，同一目标应有 600 Tick 保护期。
- 战斗只显示一套自定义 HUD。世界接口造成的永久伤痕总预算为 2048 格、每 Tick 最多 8 格，并保护返回传送门、黑曜石柱、方块实体、关键 MOD 方块和免疫标签。

## 成功与失败

### 成功

1. 在崩塌达到 100% 前造成致命伤。
2. 确认结算后祭坛开放 3×3 返回出口。
3. 进入出口后必须走原版 `showEndCredits → WinScreen → PERFORM_RESPAWN` 流程，只替换本次诗篇内容。
4. 完成诗篇确认并真正回到主世界后，视距从维度锁定值永久解锁为 16。
5. 标题页应建立结局锁：只禁用单人游戏、多人游戏和 Realms，设置、辅助功能和退出仍可用。

### 失败

1. 让崩塌先达到 100%；同 Tick 同时发生致命伤时应以失败优先。
2. 普通单人/远程客户端完成固定失败演出后，应保存、断开并正常关闭游戏。
3. 已发布 LAN 的集成服务器房主不应关闭服务器；房主回到主世界后只有自己的客户端显示方块、实体和流体为缺失材质，LAN 客人和服务端世界不受影响。
4. 失败同样建立标题页结局锁。

## F8 重置与存档隔离

1. 结局锁存在时按 F8，确认出现明确的恢复/重玩确认框。
2. 确认后应恢复 MOD 持有的桌面、记事本、窗口和资源包状态，清理客户端演出标记，并正常关闭。
3. 重启后，准确对应的本地结局存档应因 `.thefourthfrequency-corrupted` 标记显示为“存档已损坏”且不可进入。
4. 核对 `level.dat`、区块和玩家数据没有被 MOD 改写；新游戏必须使用新存档。
