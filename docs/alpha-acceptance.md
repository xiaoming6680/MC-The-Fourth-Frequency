# Alpha 0.1 人工验收与游玩说明

本文用于发布候选的人工检查。自动测试结果见 [测试策略与当前证据](testing.md)；这里的项目只有实际执行并记录环境与结果后才能标为通过。

## 前置条件

1. 安装 Minecraft 1.21.11、Fabric Loader 0.19.3、Fabric API 0.141.4+1.21.11 与 Java 21。
2. 客户端和服务器都安装 `thefourthfrequency-0.1.0-alpha.2.jar` 与 Fabric API。
3. 备份用于验收的存档；当前两个末地结局、热栏顺序变化和合法战场侵蚀都永久保留，不应在唯一正式存档上反复试路线。关闭 `mobGriefing` 不能保护末地战场。
4. 快速回归可设置 `pacing.developerAcceleration=true`。加速只缩短等待，不跳过采集、原版建筑调查、调谐、裂缝和终止钉交互。
5. 开启 Minecraft 字幕，准备 854×480 与常规窗口两种尺寸。

## 首次启动安全告示

1. 删除 `config/thefourthfrequency-safety-notice.version` 并重新打开客户端；到达标题主页面时应立即显示告示，不进入单人地图或多人服务器。
2. 确认四周完整的旧式金属终端外壳、螺钉和磨损黄铜边存在；告示外部背景与玻璃区域应保持静态，不得出现扫描线、移动扫描光、频率轨迹、边缘故障、暗角或文字重影。
3. 确认“演出范围”、“安全说明”两张左侧卡和右侧 F8 卡在 854×480 与常规窗口下清晰可读；中文句号等闭合标点不得单独占行，底部专属按钮精确显示“我已了解”。
4. 告示应明确说明 MOD 不是病毒、不会破坏电脑或个人文件，同时明确最终战会永久修改当前存档中的合法末地地形，即使 `mobGriefing=false` 也会发生；`F8` 只关闭部分游戏外演出，不能关闭权威剧情和地形后果。按 ESC 不得绕过。
5. 点击“我已了解”，确认回到原来的标题主页面且安全告示版本标记生成；随后进入单人地图和多人服务器，告示均不得再次出现。
6. 仅在验收副本中再次删除安全告示版本标记并重启客户端，确认无论之后准备进入单人还是多人，告示都先在标题主页面统一显示。

## 终端操作检查

- 手持个人终端右键：打开私人、不暂停游戏的界面；聊天中不出现终端正文。
- 点击标签或按 `1`–`4`：依次切换主页、工具、记录和文件。
- 主页：只显示当前生存目标、快捷入口和最近记录；附近信号只点亮提示，不覆盖当前目标。
- 工具页：六个工具可分别打开。选中“信号”且玩家确实处于未完成候选建筑范围时，才显示近场接收器滑条；离开范围、切换工具或关闭终端后立刻停止锁定。
- 记录页：聚合显示重要记录，不出现天气、维度切换、矿物扫描过程、内部标记和重复候选坐标，也不出现第一/第二/第三/第四频段筛选；第一次进入后全部未读标记清除。日志区滚轮只滚动事件流。
- 定位有效时：红色指北针保持正确，资源针与 `[方位]` 行给出一致的方向、距离和 Y 高度；绑定完成或读数无效时资源针停止。
- LCD：无目标时显示接收器待机；附近有目标但未打开信号工具时提示打开信号；调谐时显示信号强度、搜索、锁定中或已锁定，不显示 MHz 和固定电台名称。
- 文件页：“前人留下的文件”内固定显示“碎片1–4”，未取得的碎片保持缺失且不泄露类别名。页面首次打开没有默认悬浮/选中项；正文行左侧不显示竖条，鼠标、方向键、`Enter` 和滚轮操作保持不变。
- 波形：普通阶段保留接收波形；异常信号层出现后逐步混入心电脉冲，近场强度和锁定状态会改变稳定度。形态和颜色转换应有短过渡，不得单帧跳变或长时缓动。
- 文件详情：四个碎片集齐前显示碎片列表；集齐后可读取完整文件，长文可滚动且能返回目录，不出现密码界面。
- `Esc`：关闭界面并停止近场接收器循环音，不暂停游戏。

## 主线验收

### 1. 零号站与配发

- 新世界出生在唯一零号站内，出口可离开。
- 每位新玩家各得到一台属于自己的终端；重复加入、死亡、重连或跨维度不产生第二台有效副本。
- 站体分 Tick 完成；完成后拆除一块非关键墙并重启，方块不被自动修回。

### 2. 信号、资源与绑定

- 新终端不存在固定电台、三频校准、自动调频或全局调谐；主页从“让终端记住一个经常回来的地方”开始。
- 玩家设置住处或自然取得铁中的任一项后，终端在下一次服务端更新完成绑定；不显示绑定进度条。绑定终端不能普通丢弃或放入外部容器。
- 工具中的资源定位给出真实读数；获得定位时，指南针、高度与实际方块一致。忽略建议不阻塞主线。
- 取得铁触发现有标志异象与校正场景；绑定完成且该场景已经开始后，终端提示收到“无法归类的信号”，记录、工具与波形随之变化。它不是第四个可选择频点。
- 主页始终显示真实生存目标；玩家通过世界事件、记录、文件通知、状态灯、LCD 和方位读数理解异常变化，界面不显示隐藏组合计数。
- 满背包死亡重生时，恢复等待空槽，不覆盖或删除其他物品。

### 3. 原版建筑、碎片与共享文件

- 每个碎片生成 2–3 个真实候选坐标；同一坐标不得被两个碎片复用。候选结构来自约定池，终端卡片显示维度、X/Z、地形提示和距离，但只把文件称为“碎片1–4”。
- 每位玩家可独立选择候选作为导航目标，不改变其他玩家的指南针。到达同一碎片的任意一个候选即可调查；一个候选完成后，该碎片的其他候选停止提示。
- 进入建筑范围后终端红灯亮起，但主页任务不变。打开“工具 → 信号”后才显示近场接收器；在信号峰值 ±2 内连续保持 20 Tick，到“已锁定”后只对发现者播放短异常，再收到碎片。
- 调查不依赖箱子、战利品或特定方块。提前搜刮、爆破、改造建筑后，只要原版结构引用仍存在，就仍可完成。
- “工具 → 信号”统一显示四组匿名“未记录信号”候选，不显示第一至第四频段，也不把文件称为“碎片1–4”。调查开启前不泄露建筑名称、XZ、距离或导航日志；开启后每组只展开 2–3 个真实候选，完成后停止显示相应卡片。共享文件日志只提示去文件页查看。关键记录以标题右侧 `＋/－` 表示展开与收起；展开明细采用 12 像素行距、缩进圆点与精简坐标句式，一次只展开一条，鼠标和键盘均可操作。行动卡只描述可观察事实，不显示抽象证据名。
- 多人世界共享碎片状态。发现者保留私人行动卡和短异常；其他在线玩家以及稍后加入的终端主人收到精确提示“接收到来自【玩家名】共享的文件：《前人留下的文件 / 碎片N》”，但不收到发现者的私人行为记录。
- 集齐四个碎片后“前人留下的文件”变为完整可读文件并开启裂缝阶段。不输入命令、不寻找密码也能完成调查。

### 4. 裂缝、校正器官与身体成形

- 解锁文件后可依靠可靠读数找到并接触主世界规则裂缝。
- 趋向群按物种在不同距离停步并共同朝向目标。
- 返工体先处理让身体成形的器官，再处理携带终端或挡路玩家；会开门，只改动白名单普通方块并留下可见伤痕。旧存档已存在的设施仍可按旧规则识别，新世界不得生成五座设施。
- 三种空段都能恢复，不吞物品、不破坏非临时结构。
- 主世界↔下界真实往返后，终端记录应继续写入同一名玩家；末地或无关传送不应计入。
- 身体成形进度只在满足条件且玩家在线时前进，重连不补算离线时间；私人异象不会串给其他玩家。

### 5. 最终肉身

- 确认生命值依次触发观察（>70%）、入侵（35%–70%）、吞噬（≤35%）；随后用稳定锚治疗把生命抬回阈值之上，阶段、质量、碰撞和 Boss Bar 名称不得倒退。
- 确认稳定锚档位 8+、5–7、2–4、1、0 除既有战斗难度外，还按 `1.25 / 1.10 / 1.00 / 0.80 / 0.65` 单调改变裂解冷却；BOSS 与所有范围/地形攻击不得伤害稳定锚。
- 观察阶段裂解为 600 Tick/半径 2/最多 12 格；入侵为 440/3/24；吞噬为 280/4/36；超时坍缩为 200/4/48。操作者伤痕最多 8 格，建造者空位支架最多 3 格，采掘者三格宽沟槽最多 18 格。
- 每次世界编辑必须先有 40 Tick 地面、粒子、声音和行动栏预警。预警后普通伤害、眩晕和终止钉不得取消；实际击杀 BOSS 必须取消尚未结算的攻击。
- 在 `mobGriefing=false` 下结算一次攻击：合法末地表层仍永久改变、没有方块掉落，约四分之三为空气、四分之一为返工伤痕；成功、失败、退出重进后都不恢复。
- 验证中央 160 格之外、出口周围 8 格、基岩、黑曜石柱、哭泣黑曜石、末地传送门/折跃门/门框、负硬度块、容器、任意方块实体、稳定锚、关键 MOD 方块和 `#thefourthfrequency:end_boss_immune` 不变。
- 单场累计到 768 格后继续承受攻击：范围伤害和击退仍发生，但地形不再改变；任一 Tick 最多处理 8 个目标。
- 将 BOSS 推到 Y<32 或中央 192 格之外，确认它回收到安全地面且保留阶段、生命比例与遭遇 UUID。
- 多人加入只提高本轮见过的最大生命规模并保持生命百分比；玩家离开不得降低最大生命。无人留在末地中央时，倒计时和新攻击调度暂停。

### 6. 热栏器官错位与终止钉

- 入侵阶段进入时和每 900 Tick、吞噬进入时和每 500 Tick、坍缩进入时和每 300 Tick 触发一次 30 Tick 预警；观察阶段不触发。
- 常用武器槽优先与最远的另一个非终端槽交换；优先两个非空槽。物品数量、耐久、附魔和全部组件完全守恒，旧终端、护甲、背包主体和副手不移动。
- 两个交换槽封锁 100 Tick：从这些槽发起的攻击、使用、交互、挖掘和丢弃均被服务端拒绝，切换其他槽仍可操作。到期只解除封锁，顺序不恢复。
- 两名以上玩家按最长时间未被错位者轮转，任一时刻不得叠加两个租约。死亡、离开末地、掉线和战斗结束只清租约，不还原顺序。
- 终止钉必须通过真实实体交互生效，并兼容旧祭坛和末地 BOSS。它造成既有伤害、眩晕追击并延迟下一次攻击；对已经出现的 40 Tick 破坏预警无取消能力。

## 当前两个永久结局

每条路线使用独立存档。结算后退出并重进，确认终端余波、热栏顺序、末地伤痕和结局记录不变；旧 `undiscovered_truth` 只验收旧主世界祭坛存档兼容。

1. **阻止成功**：在七分钟归零前实际击杀 BOSS；预期 `prevention_succeeded`，身体和主动遭遇关闭，已发生的末地侵蚀保留。
2. **阻止失败**：让七分钟归零但保持 BOSS 存活；预期立即锁定 `prevention_failed`、`body_active=true` 并强制坍缩。之后击杀只关闭身体，结局不得改写。

### 8. 中文调试面板与崩坏演出

- 管理员执行 `/tff debug true` 后按 `M`，验证自适应侧栏工作台只包含总览、主线、异象和终局四个中文模块；另一名玩家未经独立授权不能打开，`F7` 不再打开面板。
- 最终怪物预警时应暂停普通追击：操作者型立刻变向，建造者型离开标记落点，采掘者型横向闪避锁定直线，周期裂解离开预测落点；四类永久编辑均为 40 Tick 预警，并有对应模型动作、粒子、音效与行动栏提示。
- 异象页应按目录顺序以可滚动行列表展示全部 16 项固定 ID，每行提供普通触发和最强触发；不出现 `disconnected_base`、`watcher_orbit`、`rework_probe`、`hostile_echo` 或复合异象入口；可单独启动、停止和恢复。
- 所有主线阶段都不得自动改写游戏图标或 Alpha 窗口标题；阶段 3 起仍可退出全屏并执行窗口几何演出，阶段 4 持续选择性缺失，阶段 5 进入高覆盖。独立「游戏窗口闪动」异象不属于固定阶段副作用，仍可单独触发和恢复。
- 首次进界崩坏后退回标题主页，左下角版本文本应精确显示 `Minecraft 1.0.0`；后续原版标题刷新不得把窗口名称改回当前版本。
- 完整退出客户端并重新启动：启动加载遮罩从第一帧到结束都应保持紫色旧版背景、旧版 Mojang 标志和旧版进度条，不得在约 50% 进度处从原版界面切换成旧版界面；结束后直接出现 Golden Days 主页、Java Alpha 图标、`Minecraft Alpha 1.0.0` 窗口标题、`Minecraft 1.0.0` 左下版本和 MOD 黄色梦核标语，不得闪现一帧原版主页或出现第二次资源加载动画。
- 重启后首次进入单人/多人世界应从第一帧起只显示正常旧版中文加载页，不再播放红色崩坏；维度切换同样不播放。

## 多人、声音与无障碍

- 至少两名真实玩家加入：碎片文件按世界共享；终端导航、私人异常、行动卡、信号未读和终端对玩家行为的描述不串台。
- 末地异象主体按长期行为与多人轮转选择目标，但不得夺取或删除终端；热栏错位只交换两个非终端槽并按租约隔离。
- 将 `peakVolume` 分别设为 0、0.4、1，确认静音、中等、峰值三档符合预期且没有持续白噪声。
- 切换 Minecraft 原生“显示字幕”选项，确认声音字幕正常显示/隐藏，且核心状态仍由文字界面表达。
- 核心信息不能只依赖颜色、波形、闪烁或声音；小窗口下折叠卡片、碎片列表、候选导航和退出路径仍可操作。

## Windows 可信 Meta

1. 保持 `meta.enabled=true`，用调试页分别启动 `window_pulse` 和 `desktop_presence`。
2. `window_pulse` 应以置中缩放动画进入闪动，结束时用反向缩放衔接，然后原样恢复 Minecraft 标题、图标、全屏、位置和尺寸；窗口始终位于当前显示器工作区。
3. `desktop_presence` 只启动一个受控记事本并打开空白的自有临时文件；随后文字应以可见的手动输入节奏逐字出现。输入助手在每个字符前校验前台窗口仍属于首次验证的记事本 PID 集，不操作任何预先存在的记事本。
4. 按 F8、断开世界或退出客户端，确认 Minecraft 窗口恢复，且只关闭本轮记录的记事本/输入助手进程并删除自有临时文件；预先打开的记事本不受影响。
5. 在演出中按 F8、退出世界和关闭客户端各检查一次立即恢复。
6. 设置 `meta.enabled=false` 后复测：只出现游戏内演出，服务端结局仍正常结算。
7. 在不同 DPI、多屏和通知设置下记录窗口恢复、通知外观与听感；这些是当前仍需目标硬件人工确认的项目。
