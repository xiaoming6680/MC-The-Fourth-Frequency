# The Fourth Frequency / 第四频段

《第四频段》是一个面向 Minecraft 1.21.11 的 Fabric 生存恐怖叙事模组。玩家从唯一的零号站醒来，借助一台会提供真实生存信息的旧终端前进，并逐渐发现：信任本身就是施工。

## 当前状态

当前开发版为 `0.2.1-beta`。生存主线、四页终端、五阶段共 16 项异象、五形态个人校正者追逐、对应维度私人镜像、“第四频段·世界接口”末地终局、原版终末之诗回程和 Windows Meta 演出均已实现。

当前数据与协议基线：

| 领域 | 当前版本 |
|---|---:|
| 世界/终端持久化 | schema v9 |
| 终端主快照 | v8 |
| 工具快照 | v4 |
| 导航载荷 | v6 |
| 异象生命周期 | v3 |
| Debug 状态 | v3 |
| 世界接口状态/协议 | format v1 / protocol v1 |

旧的四阶段“异象主体”末地战、热栏错位终局和主世界旧祭坛已从现行运行时与测试中删除；旧祭坛、旧设施和旧实体存档不再保证兼容。当前终局只使用世界接口流程。

## 运行环境

| 组件 | 版本 |
|---|---|
| Minecraft | 1.21.11 |
| Fabric Loader | 0.19.3 或更高的兼容版本 |
| Fabric API | 0.141.4+1.21.11 |
| Java | 21 |

客户端和服务器都需要安装本模组与 Fabric API。模组许可证为 [All Rights Reserved](LICENSE)。

## 安装与构建

1. 取得 `build/libs/thefourthfrequency-0.2.1-beta.jar`。
2. 将该 JAR 与对应版本的 Fabric API 放入客户端和服务器的 `mods` 目录。
3. 使用 Java 21 启动 Fabric 1.21.11，创建新世界或连入服务器。

从源码构建：

```powershell
.\gradlew.bat clean build --no-daemon
```

发布 JAR 输出到 `build/libs/`；更多命令见 [测试策略](docs/testing.md)。

## 主线与终端

首次进入世界后，零号站会按固定 Tick 预算建造；每位玩家只会获得一台有效的个人终端。手持终端右键打开私人界面，界面不暂停游戏，也不会把正文写入聊天。

- `HOME`：显示当前服务端权威生存目标、渐进式工具建议和最近记录。
- `TOOLS`：提供住处、矿物、传送门、天气、导航和要塞六个工具；未解锁项可查看条件，但服务端会拒绝越权动作。
- `RECORDS`：聚合剧情、里程碑与异象记录，进入页面即统一标记为已读。
- `FILES`：完整日记从终端创建起常驻并以乱码标题锁定；四份破损文件按世界共享发现、按终端主人独立记录阅读，全部读完后自动解锁日记。
- 数字键 `1`–`4`：切换页面；`Esc`：关闭终端。

终端的双针指南针中，红针始终指北，另一根针指向当前选定的具体矿物方块或结构。矿物由“刷新”触发 3 秒探测，并按铁/煤/金/钻石的 50%/30%/10%/10% 权重选类；玩家移动后会重新寻找该类最近的实际方块。结构候选按当前维度显示、最多三项，末地不提供候选；下界会立即提供地狱堡垒。进入结构目标水平 50 格范围后导航自动结束，已到达目标从候选中移除，终端保留一次相对方向提示。

当前任务从“认识终端”开始：依次点击顶部四个标签后完成。后续顺序为：收集 12 份任意原木/木板 → 收集 4 份铁样本 → 进入下界 → 收集 8 根烈焰棒 → 返回主世界 → 制作 4 只末影之眼 → 记录 3 次真实投掷 → 找到要塞 → 进入末地 → 阻止世界接口。每张任务卡右上角显示奖励；完成后需点击卡片手动领取，领取前手持终端会显示未读式提醒。正常主线不需要命令。

## 异象与个人追逐

异象采用五阶段滑动池。主线只提高最高阶段；实际阶段每次最多上升一级，并要求在当前阶段累计至少 20 分钟合格在线时间及完成 2 次异象。首次异象在绑定后 4–7 分钟安排，常规间隔从阶段 1 的 8–14 分钟逐步收紧到阶段 5 的 5–9 分钟；最近三次不重复，新阶段内容使用三倍权重，强接口异象另有 20–30 分钟冷却。

校正者拥有寻声者、定向者、截断者、越界者和接口校正者五个个人形态。挖矿、探索、战利品、建造和交易都能提供第一形态活动证明，不要求玩家先建家或必须通过采矿获得铁。每个形态都先经异象完成安全演示，再由终端给出行动警告，最后进入 60–110 秒正式追逐；成功追逐之间间隔 20–30 分钟，快速推进主线不会产生连续补追。

正式追逐发生在当前维度类型对应的私人校正层。入场时复制玩家周围 5×5 区块和上下各 48 格，之后按玩家所在区块持续流式扩展；水平方向没有固定 30 格限制。镜像挖掘不产出资源，简单放置物由持久账本在结束后退款。不同玩家的追逐、形态、实体和地形互相隔离，全服同时最多两场。完整规则见 [异象、终端形态与个人追逐](docs/anomalies-and-pursuits.md)。

## 末地终局：世界接口

最后一颗末影之眼会为当前存档准备末地主岛上的终局。正式战场使用原生主岛表面，不生成承载整场战斗的额外平台；只贴合地表布置 11×11 共振祭坛、20 座惰性折跃门与 10 个权威稳定锚。

仪式会冻结 1–8 名当前在线非旁观者名单；每个人都必须提交自己的有效绑定终端。完整提交前可撤回或中止，名单变化也会回滚并返还已托管终端；全员提交后事务原子生效，战斗开始。

- BOSS 最大生命为 `600 × 冻结人数`，分为“监听胚体 / 吞频构装 / 世界接口”三个只前进不后退的形态，生命阈值为 70% 和 35%。
- 世界崩坏计时为 12000 Tick（10 分钟）；只要任一冻结成员在线就会继续，全员离线时暂停。
- 稳定锚会少量治疗目标，但同时使其更慢、攻击间隔更长且承受更多伤害。每摧毁一个锚都会立即把崩坏进度向前推 600 Tick（30 秒），并进一步强化 BOSS；目标不会自行摧毁稳定锚。
- 九类权威攻击依阶段解锁：激光横扫、聚能弹、抓取砸击、精神侵入、武器托管与模拟使用、抓取投掷、热栏整槽投放、箭雨反射与强制驱逐。托管武器、投放物品和玩家控制都有持久恢复账本。
- 永久末地伤痕受 2048 格整场预算、每 Tick 8 格上限、中央 160 格范围与出口 8 格安全半径约束；基岩、黑曜石柱、传送门、方块实体、关键 MOD 方块和 `#thefourthfrequency:world_interface_immune` 均受保护。

倒计时到达 100% 的同一 Tick 先裁定失败；只有在此之前造成致命伤害才是成功。战斗后原生主岛祭坛开启 3×3 出口，玩家进入后使用真实原版 `WinScreen` 和 `PERFORM_RESPAWN` 回程，仅按成功/失败与当前语言替换该次终末之诗。

## 结局、F8 与存档

成功和失败都会写入本地结局锁。之后返回主菜单时，单人、多人和 Realms 三个游戏入口会禁用，设置与退出等其他按钮仍可用。

`F8` 不是随时可用的 Meta 开关。只有已记录结局时，按下 `F8` 才会打开完整重置确认；确认后会恢复 MOD 拥有的桌面、Notepad、窗口和资源包演出，重置客户端演出标记并正常关闭游戏。重启后，本次结局的本地单人存档会以 MOD 自有无损隔离标记显示为“存档已损坏”并禁止进入；它不会破坏 `level.dat`、区域或玩家数据。重新体验需创建新世界。

如客户端在结局事务中断，可在 JVM 参数中加入 `-Dthefourthfrequency.safeMode=true` 启动安全恢复。

失败结局的普通单人/远程客户端会在保存后正常关闭。若该客户端是已开放 LAN 的集成服务器房主，游戏不会被关闭；原版回程后仅房主客户端把方块、实体和流体渲染为缺失材质，服务器世界与 LAN 客人不受影响。

## 客户端演出与视距

第一次到达标题页时会显示 v3 安全告示，说明演出范围、设备安全、存档不可逆后果和 F8 的真实用途。确认版本单独保存在 `config/thefourthfrequency-safety-notice.version`。

客户端第一次进入世界会播放一次 Alpha 加载崩坏，之后使用持久的 Programmer Art → Golden Days Base → Golden Days Alpha 资源顺序、旧版加载界面和 `Minecraft Alpha 1.0.0` 本地演出身份。

成功终局返回主世界之前，客户端视距按维度强制锁定为主世界 3、下界 6、末地 12，设置控件不可调整，雾边界同步使用对应值。仅成功诗篇完成且玩家真正返回主世界后，视距设为 16 并持久解锁；失败不解锁。

## 配置与调试

首次启动生成 `config/thefourthfrequency.json`。

| 配置项 | 默认值 | 作用 |
|---|---:|---|
| `meta.enabled` | `true` | 允许固定白名单的本地 Meta 演出 |
| `meta.peakVolume` | `0.8` | 本模组音效峰值，校验到 0–1 |
| `pacing.developerAcceleration` | `false` | 缩短开发回归等待，不跳过真实交互 |
| `clientState.alphaDowngradeComplete` | `false` | 是否已完成首次 Alpha 降级演出 |
| `clientState.viewDistanceUnlocked` | `false` | 成功诗篇回程后是否已永久解锁视距 |

开发测试时，管理员对自己执行 `/tff debug true`，然后按可重绑的 `M` 打开中文测试工作台。当前四个模块为总览、主线、异象和终局；所有按钮动作都由服务端核对授权、稳定动作 ID、目标和当前上下文。`/tff debug false` 关闭授权。
