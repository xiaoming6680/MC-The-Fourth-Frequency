# The Fourth Frequency / 第四频段

《第四频段》是一个面向 Minecraft 1.21.11 的 Fabric 生存恐怖叙事模组。玩家从唯一的零号中转站醒来，依靠一台会提供真实资源方向的旧终端生存，并逐步发现：信任本身就是施工。

## 当前状态

Beta 0.2 的生存主线、末地祭坛终局、单人/多人规则与 Windows Meta 演出均已实现。当前开发版为 `0.2.0-beta`、终端主快照 v5、工具快照 v3、导航协议 v5、异象协议 v3、末地热栏侵入协议 v1、调试协议 v2、持久化 schema v6、末地终局记录 v3。新终局只有“阻止成功 / 阻止失败”两种有效结果；旧三结局字段只用于旧存档兼容。

既有主线与界面回归已有自动验收记录；本轮视距生命周期改造已通过四源码集编译、JUnit 139/139 与简体中文 `end-boss` 客户端套件。它仍不代表已经做完目标玩家硬件上的最终听感、美术和真实双客户端体验复核。

## 运行环境

| 组件 | 版本 |
|---|---|
| Minecraft | 1.21.11 |
| Fabric Loader | 0.19.3 或更高兼容版本 |
| Fabric API | 0.141.4+1.21.11 |
| Java | 21 |

客户端和服务器都需要安装本模组及 Fabric API。模组许可证为 [All Rights Reserved](LICENSE)。

## 安装与启动

1. 取得 `build/libs/thefourthfrequency-0.2.0-beta.jar`。
2. 将该 JAR 与对应版本的 Fabric API 放入客户端和服务器的 `mods` 目录。
3. 使用 Java 21 启动 Fabric 1.21.11，创建新世界或进入服务器。
4. 首次进入后会分 Tick 建造唯一零号站；每位玩家只会获得一台有效个人终端。

从源码构建：

```powershell
.\gradlew.bat clean build --no-daemon
```

发布 JAR 输出到 `build/libs/`。完整验证命令见 [测试策略](docs/testing.md)。

## 终端操作

手持个人终端右键打开私人界面；界面不会暂停游戏，也不会把私人文本写入聊天。

- `HOME`：显示一个真实生存任务和按当前任务与玩家状态计算的首要工具；第二项建议不会预先塞满，只有夜间缺少住处或当前目标持续没有进展时才追加。点开推荐工具会直接进入详情，左上角返回主页；只有在详情中启动“导航”或“常驻”后，主页中间才变为带关闭按钮的实时信息卡。
- `TOOLS`：提供住处、矿物、传送门、天气、导航和要塞六个工具。六项始终可见，但未解锁项只能查看条件；导航与钻石分析不会在开局立即可用。
- `RECORDS`：按时间显示全部记录，进入页面即统一标记为已读。
- `FILES`：玩家可见页名为“文件”，不使用方括号；内部继续保留稳定文件 ID。其中的《进入祭坛前》会明确警告不要拆除“稳定锚”，并要求先备份存档：最终战合法地形破坏不会复原，`mobGriefing=false` 也不能关闭。
- 数字键 `1`–`4`：切换 `HOME` / `TOOLS` / `RECORDS` / `FILES`。
- 双针指南针：红针指北；第二根针指向玩家当前选择的资源或原版建筑候选。建筑候选卡会显示维度、X/Z、地形提示和距离，并允许每位玩家独立把任一读数设为方位基准。
- 调频滑块始终保留。前期拖动只播放机械音效并改变本地波形，不发送服务端调频动作；只有导航中确有近场异常来源时，调节才可能产生玩法效果。
- 导航不会一次列出所有目的地：默认只给与当前主线最相关的一处；玩家在线且当前目标约 2 分钟没有任何进展时补充一个备选，约 5 分钟仍无进展时再开放支线；淡红闪烁的 `[不稳定信号源]` 还要求主线至少进入过下界。矿物目标和首页建议使用同一渐进规则，目标有进展就重新收束提示；已经选中的导航不会突然消失。
- `Esc`：关闭终端。

正常主线不需要命令。任务依次验证：收集 7 份任意木头（任意树种的原木或木板均可）、取得铁、进入下界、收集 6 根烈焰棒、返回主世界、制作末影之眼、记录两次投掷、抵达要塞、启动祭坛入口、进入末地并阻止最终异象主体。进入末地后，原版末影龙战斗表现由自定义 BOSS 接管；BOSS 最大生命随本轮见过的最高同时参战人数上调。塔顶装置统一称为“稳定锚”：它们会缓慢修复 BOSS，却更强地压低移动、攻击、适应频率、减伤和战场裂解频率，BOSS 不会主动摧毁它们。生命依次推进“观察 → 入侵 → 吞噬”，治疗不能倒退阶段；七分钟归零后强制进入“坍缩”，只锁定失败而不结束战斗。

异象主体会以 40 Tick 预警切开预测落点、留下冲刺伤痕、返工支架和三格宽沟槽；已预警破坏只有实际击杀 BOSS 才会取消。合法编辑只发生在末地中央 160 格，出口祭坛 8 格、基岩、黑曜石柱、传送门、容器、方块实体、稳定锚、负硬度方块、关键 MOD 方块与 `#thefourthfrequency:end_boss_immune` 始终安全。单场最多永久编辑 768 格、每 Tick 至多 8 格、无掉落，并明确忽略 `mobGriefing`。入侵阶段起还会永久交换两个非终端热栏槽并封锁 100 Tick；它不删除物品、不碰护甲、背包主体、副手或旧终端。

击杀后的 3×3 出口就在原生末地主岛祭坛上，不搭建额外平台。玩家跳入后真正调用原版末地胜利与终末之诗流程：保留 `WinScreen` 的星空、标志、滚动、音乐、跳过和重生，仅按成功/失败及当前中/英文替换诗文。读完或跳过后由原版回到主世界，再恢复战前的床或重生锚配置。

最终 BOSS 成功结算前，客户端视距按维度强制锁定：主世界 3、下界 6、末地 12，设置界面不能调整，雾边界同步采用对应距离。只有成功诗篇完成且玩家真正返回主世界后，视距才切换为 16，并持久解除所有维度的视距锁定；失败结局不会解锁。

开发测试时，由管理员对自己的玩家执行 `/tff debug true`，之后按可重绑定的 `M` 打开中文测试工作台；左侧提供总览、主线、异象和终局四个模块，异象页以可滚动行列表展示目录中的全部 16 项，并提供单异象启动和停止/恢复控制。触发回应会写入聊天历史：成功显示异象名称，失败保留服务端精确原因。`/tff debug false` 关闭；授权和调试状态按玩家独立保存。`F8` 关闭 Meta 并立即执行幂等恢复。

第一次打开游戏并到达标题主页面时，客户端会显示一次安全告示：说明恐怖演出可能延伸到游戏画面之外，明确 MOD 不是病毒、不会破坏电脑或个人文件，同时区分“电脑安全”和“存档内永久后果”——最终战会永久改变当前存档的合法末地地形，即使关闭 `mobGriefing` 也会发生。告示装入独立的旧式金属终端外壳，正文按“演出范围/安全说明/F8”分区显示；按 `F8` 只能关闭部分游戏外演出，不能关闭权威剧情或末地破坏。点击底部“我已了解”后返回标题主页面，确认只保存在本机且不再重复弹出。

客户端生命期内第一次进界会播放一次加载崩坏，然后转入旧版中文加载页。崩坏一旦触发就写入独立本地状态；以后每次启动都在 Minecraft 首次资源加载前选好 Programmer Art → Golden Days Base → Golden Days Alpha，并从启动加载遮罩的第一帧起固定使用紫色旧版背景、旧版 Mojang 标志和旧版进度条，不再在加载中途从原版界面换皮。随后直接以 Java Alpha 图标、`Minecraft Alpha 1.0.0` 窗口标题、`Minecraft 1.0.0` 菜单版本和 MOD 黄色标语进入主页，不暴露中间的原版主页，也不重播崩坏。后续进界与维度切换同样从第一帧只使用正常旧版加载页。
离开世界返回主菜单时不再替换菜单背景；本次启动选中的黄色标语会保留，`Minecraft Realms` 始终保持禁用。

## 配置

首次启动生成 `config/thefourthfrequency.json`。运行参数和两个客户端本地进度标记按分组保存在这个文件中；安全告示的当前版本确认单独保存为 `config/thefourthfrequency-safety-notice.version`。这些本地状态不会写入世界存档或服务器玩家数据。

| 配置项 | 默认值 | 作用 |
|---|---:|---|
| `meta.enabled` | `true` | 可信 Meta 启动开关；修改后需重启客户端 |
| `meta.peakVolume` | `0.8` | 本模组声音峰值，限制为 0–1 |
| `pacing.developerAcceleration` | `false` | 缩短开发回归等待，不跳过正常交互 |
| `clientState.alphaDowngradeComplete` | `false` | 客户端是否已完成一次 Alpha 降级演出 |
| `clientState.viewDistanceUnlocked` | `false` | 成功终局返回主世界后永久解除 3/6/12 视距锁；旧配置缺少此字段时安全视为 `false` |

Meta 仅在本地客户端执行固定演出；关闭它不会改变服务端主线或结局。Windows 路径只操作 Minecraft 窗口和本次启动的受控记事本：先打开空白的自有临时文件，再由自有输入助手逐字输入 MOD 内置文字。助手在每个字符前都校验前台句柄仍属于首次验证过的记事本 PID 集；不操作预先存在的窗口。断线或客户端退出会恢复临时窗口演出，并只关闭和删除本轮拥有的进程与临时文件；`F8` 仅在达成结局后用于确认完整恢复与重置。非 Windows 或 Meta 关闭时自动降级为无提示文字的游戏内伪窗口/伪记事本。
